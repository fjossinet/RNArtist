RNArtist databases

Quickstart

  • create a new folder on your computer
  • create one or several folders corresponding to different projects or RNA families
  • in each subfolder, add some data (vienna files)
  • choose the root folder as your new RNArtist database with this button
  • once the database loaded, select a subfolder in the tree to compute and load the 2Ds in the database explorer

The structure of an RNArtist database

An RNArtist database is simply a folder containing subfolders with structural data (vienna/ct/bpseq/pdb/stockholm formatted files).

When you choose a folder with this button, RNArtist will do the following tasks:

  • create an index file named ".rnartist_db_index" in this folder
  • search for all subfolders containing structural data (vienna/bpseq/ct/stockholm formatted files). Each subfolder is recorded in the index file
  • for each folder recorded, an RNArtist script file is created at the same level, with the same name and with a "kts" suffix.

In an RNArtist database, you will find structural files (vienna/bpseq/ct/stockholm formatted files), RNArtist scripts (file whose name end with "kts") and PNG files storing a preview of the 2D drawing for a given structural file.

Scripts

Scripts store the instructions to plot an RNA 2D (theme, layout, input/output details,...). An RNArtist database contains scripts with two slightly different roles:

  • RNArtist scripts with the same name than folders: they contains the default instructions to plot the first draft of any structure stored in this folder. They are used when you load a folder by clicking on it in the database explorer panel
  • RNArtist scripts with the same name than vienna/bpseq/ct/stockholm formatted files: they contain the instructions to plot the last version of the 2D described in the structural file.

Drawings

RNArtist creates a folder named "drawings" to store as PNG files previews for each RNA 2D that has been loaded with RNArtist.
